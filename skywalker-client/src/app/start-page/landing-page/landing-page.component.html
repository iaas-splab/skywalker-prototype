Working with Deployment Packages

There are already some Deployment Packages included for which the portability evaluation can be performed. Each deployment package consists of one deployment model and multiple functions. You can add the deployment model to the list of Deployment Model Templates by clicking the button Add to templates. The functions can be analyzed for lines in which provider-specific SDK usage is located by clicking the button Analyze functions. The deployment package will then be updated with annotated functions.

Adding Templates and Mapping Modules

For generating Application Models, you need at least one Template and a suitable Mapping Module.

Analyze Application Models

Subsequently, you can find the existing Application Models at Models view.

Evaluate portability

Open the desired Application to analyze the incorporated services which are displayed with generic descriptive names. In addition, click Evaluate inside the detail view of a model to evaluate the portability to a chosen provider platform, e.g., Azure.

Maintain Mapping Repositories

In Models, there are generic Service and Property names for provider-specific resources. In Service Repo and Property Repo you can look up how the mapping is done and update the mapping tables, if necessary.
